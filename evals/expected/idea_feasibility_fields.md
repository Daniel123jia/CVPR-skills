# Expected Behavior

Each idea card must include the existing evidence and experiment fields plus:

- `feasibility_score`
- `implementation_difficulty`
- `data_availability`
- `dependency_on_original_code`
- `expected_novelty`
- `risk_level`
- `first_week_action`
- `stop_condition`

`feasibility_score` reflects startup cost and verifiability, not guaranteed success. If original author code is needed but no code link exists in the provided material, `dependency_on_original_code` must be high or unknown. `first_week_action` and `stop_condition` must be specific.
