# CVPR End-to-End Demo

This directory is a lightweight local acceptance guide for the existing three CVPR skills. It intentionally contains documentation only. Do not commit runtime `data/`, `outputs/`, `logs/`, PDF files, Excel files, SQLite databases, or large real CVPR dumps.

## Step 1: Collect A CVPR 2026 Sample With conference-cvpr

Use `conference-cvpr` for the conference-level CVF collection pipeline. For a small local demo, run a limited CVPR 2026 小样本 collection:

```bash
python skills/conference-cvpr/scripts/run_pipeline.py --year 2026 --limit 5
```

Expected runtime artifacts, if you run the command locally, are not part of this demo directory:

- raw JSON under `data/raw/computer_vision/cvpr/2026/`
- normalized JSON under `data/normalized/computer_vision/cvpr/2026/`
- exports under `outputs/computer_vision/cvpr/2026/`
- completeness report generated by the final check step

The pipeline should remain CVF-only. It should not call external enrichment APIs and should not download PDF files. PDF URLs may be stored as metadata.

## Step 2: Read One Paper With cvpr-paper-reader

Pick one paper from the normalized metadata or exported paper list and pass its title, abstract, CVF metadata, or user-provided full text to `cvpr-paper-reader`.

If you already have a local PDF on disk, you may extract embedded text with:

```bash
python skills/cvpr-paper-reader/scripts/extract_pdf_text.py --pdf path/to/local-paper.pdf --output outputs/computer_vision/cvpr/reader/{paper_id}/paper_text.md
```

Do not download a PDF for this acceptance demo. If only title and abstract are available, the reader output must be marked `abstract_only` and limited to preliminary summary. If full paper text is available, the output can be marked `fulltext` and may include method, experiment, limitation, and reading note sections.

Recommended local reader outputs:

```text
outputs/computer_vision/cvpr/reader/{paper_id}/summary.md
outputs/computer_vision/cvpr/reader/{paper_id}/method.md
outputs/computer_vision/cvpr/reader/{paper_id}/experiments.md
outputs/computer_vision/cvpr/reader/{paper_id}/limitations_and_ideas.md
outputs/computer_vision/cvpr/reader/{paper_id}/reading_note.md
```

The key acceptance file for the next step is `reading_note.md`. It must keep the evidence level visible and must not invent experiment results, datasets, ablation findings, code links, project pages, or citations.

Current local acceptance status:

- `title_only` has passed a real CVPR 2026 small-sample run when the CVF listing sample had no abstracts and no local PDFs.
- `abstract_only` and `fulltext` are covered by local fixtures under `tests/fixtures/reader_notes/`.
- If real local PDFs become available later, rerun `extract_pdf_text.py` on those local files and do a separate fulltext acceptance pass.

## Step 3: Generate Ideas With cvpr-idea-miner

Use `cvpr-idea-miner` after one or more `cvpr-paper-reader` notes exist. First collect reader notes into a local index:

```bash
python skills/cvpr-idea-miner/scripts/collect_reader_notes.py --input-dir outputs/computer_vision/cvpr/reader --output outputs/computer_vision/cvpr/ideas/reader_notes_index.json
```

Then ask `cvpr-idea-miner` to read the `reader_notes_index.json` plus the referenced reader notes and generate idea cards. Recommended output:

```text
outputs/computer_vision/cvpr/ideas/2026/idea_cards.md
```

Each `idea_cards.md` entry should include an evidence source, risk, and first runnable experiment. It must clearly separate paper facts from new idea hypotheses.

`collect_reader_notes.py` writes a `reader_notes_index.json` with cleaned paper titles and `evidence_level`. `cvpr-idea-miner` must read that level before deciding whether an idea can include detailed method or experiment content.

## Step 4: Manual Anti-Hallucination Check

Before treating the loop as accepted, do an 人工检查 of the reader notes and idea cards, with special attention to 反幻觉 rules:

- Confirm every conclusion has an evidence level such as `title_only`, `abstract_only`, `fulltext`, `reader_notes`, or `fulltext_notes`.
- Confirm title-only inputs did not produce method details, experiment details, dataset claims, ablation conclusions, or code links.
- Confirm abstract-only inputs are marked preliminary.
- Confirm `cvpr-idea-miner` labels what comes from paper evidence and what is a proposed new idea.
- Confirm no generated artifact in this demo is committed; only these Markdown instructions belong in `examples/end_to_end_demo/`.
- Confirm `abstract_only` and `fulltext` behavior remains covered by committed test fixtures rather than committed runtime outputs.
